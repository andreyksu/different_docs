Python

Виды пространств имен:
	Множество встроенных имен.
	Глобальные имен в модуле.
	Локальные имен при вызове функции.
	Множество атрибутов объекта, так же образует пространство имен.

Модуль вводит пространство ИМЕН.
	Имена в модулях, являются атрибутами модуля.
	По этому можно сказать Атрибуты модуля соотносятся с Глобальными именами модуля (они совместно используют одно пространство имен).



Пространство имен содерждащее встроенные имена, создается при запуске интерпертатора и существует все время его работы.
	Модуль __builtin__.
Глобальное пространство имен модуля, создается когда он считывается и как правило живет до момента завершения работы интерпертатора.
	Рассматриваются как часть модуля __mani__ (для интерактивных).
Локальное пространство имен:
	Создается при вызове функции и уничтожается при завершении фукнции.
	
	
Порядок поиска:
	Локальное пространство (меод, класс) → Глобальное (модуль)
	
В Python ВСЕГда присваивание ведется к локальной переменной, если не указано global.

При определении класса - создается новое пространство имен.
	Используетсяд в качестве локальной области.
	
	
x = MyClass()
x.f()

А можно MyClass.f(x) - вот он self

Атрибуты данных при совпадении имен переписывают/перебивают методы с теми же именами. Речь про классы.
